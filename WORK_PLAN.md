# 將棋單機遊戲工作計畫

## 專案目標
建立可離線執行的將棋遊戲，完整支援標準規則，提供人類對電腦及本機雙人對戰，並具備日式風格視覺設計。

## 範圍假設
- 平台：桌面或本機瀏覽器執行
- 介面：2D 棋盤與日式風格 UI
- AI：先可玩，之後再提升強度
- 語言：先以可執行骨架為主，後續再優化

## 待辦事項（SOP）
- [ ] 確認目標平台與技術棧
- [ ] 定義規則覆蓋範圍（升變、打入、千日手、入玉等）
- [ ] 建立專案骨架（建置、執行、基本 UI 殼）
- [ ] 建立核心資料模型（棋盤、棋子、走法）
- [ ] 實作走法產生（合法走子、升變判定）
- [ ] 實作打入規則（含二步、打步詰）
- [ ] 實作狀態轉換（走子、還原）
- [ ] 實作規則驗證（王手、將死、非法走法）
- [ ] 實作千日手與裁定
- [ ] 實作勝負條件（將死、投了、和局、入玉）
- [ ] UI 棋盤與棋子繪製
- [ ] 輸入操作（點擊/拖曳、提示）
- [ ] 日式視覺風格（字體、材質、色彩）
- [ ] 本機雙人模式
- [ ] AI v1（基礎評估 + 搜尋）
- [ ] AI 難度分級
- [ ] 走譜紀錄、悔棋
- [ ] 存檔/讀檔
- [ ] 音效（選配）
- [ ] 設定與選項面板
- [ ] 教學/說明
- [ ] 規則與邊界測試
- [ ] QA 全面驗證
- [ ] 打包/發佈

## 里程碑
- M1：核心規則與合法走子
- M2：可玩 UI + 本機雙人
- M3：AI v1 可對戰
- M4：視覺完善 + 規則邊界
- M5：可發佈版本

## 風險與備註
- 規則完整性需要大量測試（打入、千日手、入玉）
- AI 強度取決於評估函數與搜尋深度
- 視覺素材需留意授權或自製
